# LieAlgebra Crypto

**Пост-квантовая криптосистема на основе универсальной обёртывающей алгебры Ли**

---

## О программе

LieAlgebra Crypto — это программная реализация симметричной криптосистемы, основанной на универсальной обёртывающей алгебре конечномерной нильпотентной алгебры Ли над полем GF(257). Система использует некоммутативность алгебраических структур для обеспечения криптографической стойкости, что делает её перспективным кандидатом для пост-квантовой криптографии.

### Основные возможности

- **Автоматический выбор размерности** — программа анализирует выбранный текст и автоматически определяет оптимальную размерность алгебры Ли.
- **Гибкое управление ключами** — ключ представляет собой обычную текстовую строку (пароль), которая может быть введена вручную или сгенерирована автоматически.
- **Сохранение компонентов** — алгебра, ключ и шифротекст сохраняются в отдельные JSON-файлы для удобного обмена.
- **Интуитивный интерфейс** — вкладки для шифрования, управления алгеброй, ключом и дешифрования делают работу простой и понятной.
- **Детерминированная генерация** — один и тот же пароль всегда генерирует один и тот же ключ, что обеспечивает возможность дешифрования.

## Теоретическая основа

### Алгебры Ли

Алгебра Ли L над полем k — это векторное пространство с билинейной операцией [·,·], удовлетворяющей:

- антисимметричности: [x,x] = 0, [x,y] = −[y,x];
- тождеству Якоби: [x,[y,z]] + [y,[z,x]] + [z,[x,y]] = 0.

### Нильпотентные алгебры Ли

Алгебра Ли называется нильпотентной ступени s, если любой коммутатор длины s+1 равен нулю. В программе генерируются случайные нильпотентные алгебры, где [x_i, x_j] может быть ненулевым только для k > j и k ≤ j + step.

### Универсальная обёртывающая алгебра U(L)

Для алгебры Ли L её универсальная обёртывающая алгебра U(L) определяется как фактор тензорной алгебры T(L) по двустороннему идеалу, порождённому элементами x⊗y − y⊗x − [x,y]. В U(L) выполнены соотношения xy − yx = [x,y].

### Теорема Пуанкаре–Биркгофа–Витта (PBW)

Согласно теореме PBW, множество упорядоченных мономов x₁^{k₁}x₂^{k₂}…xₙ^{kₙ} (k_i ≥ 0) образует базис U(L) как векторного пространства. Это позволяет однозначно представлять любые элементы в виде линейной комбинации упорядоченных мономов и нормализовать полиномы.

### Базисы Грёбнера–Ширшова

В U(L) соотношения x_j x_i = x_i x_j + [x_j,x_i] (для j>i) задают систему переписывающих правил. Стандартный базис (базис Грёбнера–Ширшова) позволяет эффективно приводить любой элемент к нормальной форме. Для каждой пары (i,j) с i>j строится правило редукции: x_i x_j → x_j x_i + Σ c_{ij}^k x_k.

### Конечное поле GF(257)

Все вычисления производятся в поле характеристики 257. Число 257 простое, поэтому для любого ненулевого элемента a существует обратный a⁻¹ ≡ a²⁵⁵ (mod 257). Выбор поля обусловлен близостью к степени двойки (256), что удобно для кодирования байтов.

## Установка и запуск

### Системные требования

- Операционная система: Windows / Linux / macOS
- Python 3.6 или выше
- Библиотека python-docx (только для генерации отчётов)

### Установка

1. Скачайте папку с программой или склонируйте репозиторий:
   ```bash
   git clone https://github.com/zheyas/Li_Shifr_with_gui.git
   cd liealgebra-crypto
   ```

2. Установите необходимые зависимости (если требуется работа с документацией):
   ```bash
   pip install python-docx
   ```

3. Запустите программу:
   ```bash
   python gui.py
   ```

### Структура файлов

- `gui.py` — главный файл с графическим интерфейсом
- `gf257.py` — реализация конечного поля GF(257)
- `monom.py` — классы для работы с мономами
- `polynomial.py` — классы для работы с полиномами
- `lie_algebra.py` — генерация нильпотентных алгебр Ли
- `multiplication.py` — правила умножения и кэширование
- `encoding.py` — кодирование текста в полиномы
- `serialization.py` — сериализация полиномов в бинарный формат
- `file_io.py` — сохранение и загрузка JSON-файлов

## Руководство пользователя

### Основной рабочий процесс

1. **Выбор текста для шифрования**
   - Перейдите на вкладку «Шифрование»
   - Нажмите «Выбрать и анализировать» и укажите текстовый файл
   - Программа определит размер текста и требуемую размерность алгебры

2. **Генерация алгебры Ли**
   - Перейдите на вкладку «Алгебра Ли»
   - Установите параметры (можно использовать рекомендуемую размерность)
   - Нажмите «Сгенерировать» или «Загрузить» готовую алгебру

3. **Создание ключа**
   - Перейдите на вкладку «Ключ»
   - Введите пароль вручную или нажмите «Сгенерировать случайный ключ»
   - Ключ автоматически преобразуется в полином

4. **Шифрование**
   - Вернитесь на вкладку «Шифрование»
   - Когда все компоненты готовы, кнопка шифрования станет активной
   - Нажмите «Выполнить шифрование» — шифротекст сохранится в `cipher.json`

5. **Дешифрование**
   - Перейдите на вкладку «Дешифрование»
   - Выберите файлы алгебры, шифротекста и укажите выходной файл
   - Введите тот же пароль или загрузите ключ из файла
   - Нажмите «Выполнить дешифрование»

### Форматы файлов

- **Алгебра** (`algebra.json`) — содержит размерность, ступень, плотность и структурные константы
- **Ключ** (`key.json`) — содержит размерность и base64-кодированный полином
- **Шифротекст** (`cipher.json`) — содержит размерность и base64-кодированный полином-шифротекст

## Научная новизна

- Автоматическое определение оптимальной размерности алгебры Ли на основе длины текста
- Динамическое распределение переменных для кодирования позиции с адаптацией к текстам произвольной длины
- Реализация алгоритма деления с остатком с использованием кучи (сложность O(t log t))
- Кэширование результатов умножения мономов для ускорения многократных операций
- Применение теории базисов Грёбнера–Ширшова для обоснования корректности дешифрования

## Преимущества

- **Потенциальная квантовая устойчивость** — для некоммутативных алгебраических структур неизвестны эффективные квантовые алгоритмы
- **Гибкость параметров** — возможность регулировать сложность системы изменением размерности, ступени нильпотентности и плотности структурных констант
- **Математическая обоснованность** — опора на фундаментальные теоремы (PBW, базисы Грёбнера–Ширшова)
- **Автоматическая адаптация** — к длине сообщения без необходимости ручной настройки
- **Простота использования** — интуитивный интерфейс, не требующий специальных математических знаний

- **Применение в стеганографии** — скрытое встраивание информации в мультимедийные данные через геометрические преобразования
