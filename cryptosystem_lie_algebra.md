## Введение

Данная работа представляет собой реализацию криптосистемы, основанной на универсальных обёртывающих алгебрах конечномерных нильпотентных алгебр Ли. Идея восходит к работам В.Н. Латышева [1, 2], в которых предлагается использовать некоммутативные алгебры со строгой фильтрацией для построения криптографических примитивов. В отличие от классических схем (RSA, эллиптическая криптография), данная система относится к перспективному направлению **пост-квантовой криптографии**, поскольку её стойкость основывается на сложности задач в некоммутативных структурах, для которых неизвестны эффективные квантовые алгоритмы.

**Ключевые особенности данной реализации:**
- Автоматическое определение оптимальной размерности алгебры Ли на основе длины текста
- Динамическое распределение переменных для кодирования позиции
- Поддержка текстов произвольной длины (до 2^(4*(dim-1)) байт)
- Сохранение всех параметров в JSON для возможности дешифрования

## Математические основы

### 2.1 Конечное поле GF(257)

Все вычисления производятся в поле характеристики 257. Число 257 простое, что позволяет использовать арифметику по модулю 257. Обратный элемент вычисляется по формуле:

$$a^{-1} \equiv a^{255} \pmod{257},$$

что следует из малой теоремы Ферма. Выбор поля 257 обусловлен тем, что оно близко к степени двойки (256) и позволяет эффективно кодировать байты данных.

### 2.2 Алгебры Ли

Алгебра Ли $L$ над полем $k$ — это векторное пространство с билинейной операцией $[\cdot,\cdot]: L\times L\to L$, удовлетворяющей:

- антисимметричности: $[x,x]=0$, $[x,y]=-[y,x]$;
- тождеству Якоби: $[x,[y,z]]+[y,[z,x]]+[z,[x,y]]=0$.

Алгебра называется **нильпотентной ступени $s$**, если любой коммутатор длины $s+1$ равен нулю. Например, для $s=2$ имеем $[[x,y],z]=0$ для всех $x,y,z\in L$. В данной реализации ступень нильпотентности задаётся параметром `step` и определяет, как далеко могут простираться коммутаторы.

### 2.3 Универсальная обёртывающая алгебра

Для алгебры Ли $L$ её универсальная обёртывающая алгебра $U(L)$ определяется как фактор тензорной алгебры $T(L)$ по двустороннему идеалу, порождённому элементами $x\otimes y - y\otimes x - [x,y]$. Это ассоциативная алгебра с единицей, в которой выполнены соотношения:

$$xy - yx = [x,y] \quad \forall x,y\in L.$$

Универсальная обёртывающая алгебра играет ключевую роль, поскольку именно в ней выполняются все умножения при шифровании и дешифровании.

### 2.4 Теорема Пуанкаре–Биркгофа–Витта

**Теорема (PBW).** Если $\{x_1,\dots,x_n\}$ — базис $L$, то множество мономов

$$x_1^{k_1}x_2^{k_2}\dots x_n^{k_n}, \quad k_i\ge 0,$$

образует базис $U(L)$ как векторного пространства. Это означает, что любой элемент $U(L)$ однозначно представим в виде линейной комбинации упорядоченных мономов (порядок индексов фиксирован). На этой теореме основана возможность нормализации полиномов в нашей реализации.

### 2.5 Базисы Грёбнера–Ширшова

В контексте ассоциативных алгебр, заданных образующими и соотношениями, **стандартный базис (базис Грёбнера–Ширшова)** идеала соотношений позволяет эффективно приводить любой элемент к нормальной форме. В нашем случае соотношения имеют вид $x_jx_i = x_ix_j + [x_j,x_i]$ для $j>i$. Эти соотношения задают правила приведения произвольного монома к линейной комбинации упорядоченных мономов.

## Описание криптосистемы

### 3.1 Генерация алгебры Ли

На первом этапе генерируется случайная нильпотентная алгебра Ли заданной размерности $n$ и ступени $s$. Для каждой пары индексов $i<j$ с вероятностью `density` выбираются ненулевые структурные константы $c_{ij}^k$ для $k$ от $j+1$ до $j+s$ (гарантируется нильпотентность). Таким образом:

$$[x_i, x_j] = \sum_{k=j+1}^{\min(j+s, n)-1} c_{ij}^k x_k.$$

Антисимметричность автоматически даёт значения для $[x_j, x_i]$.

### 3.2 Кодирование текста в полином

Текст преобразуется в последовательность байт (UTF-8). Каждый байт представляется отдельным мономом. Схема кодирования динамически адаптируется под размерность алгебры:

- **Переменная для данных:** всегда используется первая переменная ($x_0$), в показатель степени которой записывается значение байта (0–255).
- **Переменные для позиции:** используется максимально возможное количество переменных из оставшихся (от $x_1$ до $x_{n-1}$). Каждая переменная хранит 4 бита позиции.

Таким образом, если доступно $p$ переменных для позиции, максимальная длина текста составляет $2^{4p} - 1$ байт.

Дополнительно в полином добавляется **маркерный моном**, кодирующий длину сообщения (также в переменных позиции) с коэффициентом 1.

### 3.3 Автоматическое определение размерности

Одной из ключевых особенностей реализации является автоматический выбор размерности алгебры Ли на основе длины текста:

1. Вычисляется длина текста в байтах $L$
2. Определяется необходимое количество бит для позиции: $b = \lceil \log_2(L) \rceil + 1$
3. Вычисляется необходимое количество переменных для позиции: $p = \lceil b/4 \rceil$
4. Итоговая размерность: $n = p + 2$ (одна переменная для данных + одна резервная)

Это гарантирует, что размерность всегда достаточна для кодирования текста любой длины.

### 3.4 Генерация ключа

Ключ $K$ — это случайный полином из нескольких мономов (обычно 2-3). Каждый моном имеет небольшую полную степень (1-2) и случайные показатели, распределённые по всем переменным. Коэффициенты выбираются из $GF(257)$. Для ускорения вычислений степень ключа минимизируется.

### 3.5 Шифрование

Шифротекст $C$ получается умножением полинома сообщения $M$ на ключ $K$ **справа**:

$$C = M \cdot K.$$

Умножение выполняется с использованием правил редукции, построенных по алгебре Ли. Результат $C$ — полином, как правило, с бóльшим числом мономов, чем $M$. Все промежуточные результаты кэшируются для ускорения повторяющихся операций.

### 3.6 Дешифрование

Получатель, зная ключ $K$, выполняет левое деление $C$ на $K$:

$$C = Q \cdot K + R.$$

Благодаря тому, что $\{K\}$ является стандартным базисом порождённого им левого идеала, остаток $R$ всегда равен нулю, а частное $Q$ в точности совпадает с исходным $M$. Алгоритм деления использует кучу для быстрого доступа к ведущему члену, что даёт сложность $O(t\log t)$, где $t$ — число мономов в $C$.

После получения $Q$ из него извлекаются позиции и байты. Количество переменных для позиции определяется автоматически на основе анализа максимальной позиции в восстановленных данных.

### 4.2 Класс `GF257`

```python
class GF257:
    """
    Элемент конечного поля GF(257).
    Все операции выполняются по модулю 257.
    """
    def __init__(self, val):
        self.val = val % 257

    def __add__(self, other):
        return GF257(self.val + other.val)

    def __sub__(self, other):
        return GF257(self.val - other.val)

    def __mul__(self, other):
        return GF257(self.val * other.val)

    def __truediv__(self, other):
        return self * other.inv()

    def inv(self):
        return GF257(pow(self.val, 255, 257))

    def __eq__(self, other):
        return self.val == other.val

    def __lt__(self, other):
        return self.val < other.val
```

### 4.3 Генерация алгебры Ли (`RandomLieAlgebra`)

```python
class RandomLieAlgebra:
    """
    Генератор случайной нильпотентной алгебры Ли.

    Атрибуты:
        dim (int): размерность алгебры
        step (int): ступень нильпотентности
        density (float): вероятность ненулевого структурного элемента
        structure (dict): словарь структурных констант
    """
    def __init__(self, dim, step, density=0.3):
        self.dim = dim
        self.step = step
        self.density = density
        self.structure = {}
        self._generate()

    def _generate(self):
        for i in range(self.dim):
            for j in range(i+1, self.dim):
                terms = []
                max_k = min(j+self.step, self.dim)
                for k in range(j+1, max_k):
                    if random.random() < self.density:
                        coeff = GF257(random.randint(1,256))
                        terms.append((k, coeff))
                if terms:
                    self.structure[(i,j)] = terms
                    self.structure[(j,i)] = [(k, -coeff) for (k,coeff) in terms]

    def get_reduction_rules(self):
        """Построение правил редукции для универсальной обёртывающей."""
        rules = {}
        for (i,j), terms in self.structure.items():
            if i > j:
                rule_terms = []
                mon_swap = [0]*self.dim
                mon_swap[j] = 1
                mon_swap[i] = 1
                rule_terms.append((tuple(mon_swap), GF1))
                for k, coeff in terms:
                    mon_comm = [0]*self.dim
                    mon_comm[k] = 1
                    rule_terms.append((tuple(mon_comm), coeff))
                rules[(i,j)] = rule_terms
        return rules
```

### 4.4 Мономы и полиномы (`Monom`, `Polynomial`)

**Моном** хранит кортеж показателей степени и размерность. Для него определён порядок, используемый в умножении: сначала по убыванию полной степени, затем лексикографически по убыванию показателей.

```python
@total_ordering
class Monom:
    def __init__(self, exps, dim):
        self.exps = tuple(exps)
        self.dim = dim

    def __lt__(self, other):
        if sum(self.exps) != sum(other.exps):
            return sum(self.exps) > sum(other.exps)
        return self.exps > other.exps

    def can_divide(self, other):
        return all(self.exps[i] >= other.exps[i] for i in range(self.dim))

    def div(self, other):
        return Monom([self.exps[i]-other.exps[i] for i in range(self.dim)], self.dim)
```

**Полином** хранит свои члены в отсортированном списке `_terms` (кортежи `(monom, coeff)`). Благодаря бинарному поиску при добавлении, список всегда остаётся отсортированным.

```python
class Polynomial:
    def __init__(self, terms=None, dim=6):
        self.dim = dim
        self._terms = []  # отсортированный список (monom, coeff)
        if terms:
            for mon, coeff in terms.items():
                self._add_term(mon, coeff)

    def _add_term(self, mon, coeff):
        if coeff == GF0:
            return
        lo, hi = 0, len(self._terms)
        while lo < hi:
            mid = (lo+hi)//2
            if self._terms[mid][0] == mon:
                new_coeff = self._terms[mid][1] + coeff
                if new_coeff == GF0:
                    self._terms.pop(mid)
                else:
                    self._terms[mid] = (mon, new_coeff)
                return
            elif self._terms[mid][0] < mon:
                lo = mid+1
            else:
                hi = mid
        self._terms.insert(lo, (mon, coeff))

        return self._terms[0] if self._terms else (None, GF0)
```

### 4.5 Правила умножения и редукция

```python
_reduction_rules = {}
_monom_mul_cache = {}
_monom_mul_cache_hits = 0
_monom_mul_cache_misses = 0

def multiply_monoms(m1, m2, dim):
    """Умножение двух мономов с кэшированием."""
    key = (m1.exps, m2.exps)
    if key in _monom_mul_cache:
        return _monom_mul_cache[key]
    # преобразуем мономы в списки переменных
    vars1 = [i for i,exp in enumerate(m1.exps) for _ in range(exp)]
    vars2 = [i for i,exp in enumerate(m2.exps) for _ in range(exp)]
    result = _sort_vars(vars1+vars2, dim)
    _monom_mul_cache[key] = result
    return result

def _sort_vars(vars_list, dim, depth=0):
    """Приведение списка переменных к упорядоченному виду с применением правил."""
    if len(vars_list) <= 1:
        exps = [0]*dim
        for v in vars_list:
            exps[v] += 1
        return Polynomial({Monom(exps,dim): GF1}, dim)
    for idx in range(len(vars_list)-1):
        i, j = vars_list[idx], vars_list[idx+1]
        if i > j:
            rule = get_rule(i,j)
            if rule is None:
                new_list = vars_list[:idx] + [j,i] + vars_list[idx+2:]
                return _sort_vars(new_list, dim, depth+1)
            else:
                res = Polynomial(dim=dim)
                for mon_exps, coeff in rule:
                    rule_vars = []
                    for k,exp in enumerate(mon_exps):
                        rule_vars.extend([k]*exp)
                    new_list = vars_list[:idx] + rule_vars + vars_list[idx+2:]
                    term = _sort_vars(new_list, dim, depth+1)
                    res = res + (term * coeff)
                return res
    exps = [0]*dim
    for v in vars_list:
        exps[v] += 1
    return Polynomial({Monom(exps,dim): GF1}, dim)
```

### 4.6 Сериализация полиномов

Для сохранения и загрузки полиномов используется бинарный формат:

```python
def polynomial_to_bytes(poly):
    """Сериализует полином в бинарный формат."""
    data = bytearray()
    data.extend(struct.pack('<I', len(poly._terms)))
    for mon, coeff in poly._terms:
        for exp in mon.exps:
            data.extend(struct.pack('<I', exp))
        data.extend(struct.pack('<i', coeff.val))
    return bytes(data)

def bytes_to_polynomial(data, dim):
    """Десериализует полином из бинарных данных."""
    terms = {}
    offset = 0
    num_terms = struct.unpack('<I', data[offset:offset+4])[0]
    offset += 4
    for _ in range(num_terms):
        exps = []
        for i in range(dim):
            exp = struct.unpack('<I', data[offset:offset+4])[0]
            offset += 4
            exps.append(exp)
        coeff_val = struct.unpack('<i', data[offset:offset+4])[0]
        offset += 4
        mon = Monom(exps, dim)
        terms[mon] = GF257(coeff_val)
    return Polynomial(terms, dim)
```

### 4.7 Автоматическое определение размерности

```python
def calculate_required_dim(text_length):
    """
    Автоматически вычисляет необходимую размерность алгебры Ли.

    Аргументы:
        text_length (int): Длина текста в байтах

    Возвращает:
        int: Оптимальная размерность для кодирования
    """
    # Сколько бит нужно для кодирования позиции
    position_bits_needed = math.ceil(math.log2(text_length)) + 1

    # Сколько переменных нужно для позиции (по 4 бита)
    pos_vars_needed = math.ceil(position_bits_needed / 4)

    # Добавляем 1 переменную для данных и небольшой запас
    recommended_dim = pos_vars_needed + 1 + 1

    # Минимальная размерность - 4
    recommended_dim = max(4, recommended_dim)

    return recommended_dim
```

### 4.8 Динамическое кодирование позиции

В `encrypt.py` функция `text_to_polynomial` динамически определяет, сколько переменных использовать для позиции:

```python
def text_to_polynomial(text, dim):
    data = text.encode('utf-8')
    poly = Polynomial(dim=dim)

    # Сколько бит нужно для позиции
    position_bits_needed = math.ceil(math.log2(len(data))) + 1

    # Сколько переменных нужно для позиции
    pos_vars_needed = math.ceil(position_bits_needed / 4)
    pos_vars_count = min(pos_vars_needed, dim - 1)

    # Переменные для позиции (последние pos_vars_count)
    pos_vars = list(range(dim - pos_vars_count, dim))
    # Переменные для данных (остальные)
    data_vars = list(range(dim - pos_vars_count))

    for i, byte in enumerate(data):
        exps = [0] * dim
        # Кодируем байт в первую переменную данных
        if data_vars:
            exps[data_vars[0]] = byte
        # Кодируем позицию
        pos = i
        for idx in pos_vars:
            exps[idx] = pos & 15
            pos >>= 4
        poly._add_term(Monom(exps, dim), GF1)

    # Маркер конца
    marker_exps = [0] * dim
    pos = len(data)
    for idx in pos_vars:
        marker_exps[idx] = pos & 15
        pos >>= 4
    poly._add_term(Monom(marker_exps, dim), GF1)

    return poly
```

В `decrypt.py` функция `polynomial_to_bytes` автоматически определяет, сколько переменных использовалось для позиции, анализируя максимальную позицию в данных.

### 4.9 Генерация ключа

```python
def random_key(dim, num_terms=2, max_coeff=5, max_degree=1, seed=42):
    """
    Генерирует случайный полином-ключ.

    Для ускорения вычислений используется минимальная степень (1)
    и малое количество мономов (2).
    """
    if seed is not None:
        random.seed(seed)
    terms = {}
    for _ in range(num_terms):
        total_degree = random.randint(1, max_degree)
        exps = [0]*dim
        for _ in range(total_degree):
            idx = random.randint(0, dim-1)
            exps[idx] += 1
        coeff = GF257(random.randint(1, max_coeff))
        terms[Monom(exps, dim)] = coeff
    return Polynomial(terms, dim)
```

### 4.10 Основная программа `encrypt.py`

```python
def main():
   # Чтение текста для определения размерности
   with open("../text.txt", "r", encoding="utf-8") as f:
      text = f.read()

   # Автоматическое определение размерности
   data_bytes = text.encode('utf-8')
   dim = calculate_required_dim(len(data_bytes))

   # Генерация алгебры Ли
   lie = RandomLieAlgebra(dim=dim, step=6, density=0.3)
   rules = lie.get_reduction_rules()
   set_rules(rules)

   # Генерация ключа
   key = random_key(dim=dim, num_terms=2, max_degree=1)

   # Кодирование и шифрование
   M = text_to_polynomial(text, dim)
   C = M * key

   # Сохранение в JSON
   exchange_data = {
      "version": 4,
      "dim": dim,
      "step": 6,
      "density": 0.3,
      "ciphertext": base64.b64encode(polynomial_to_bytes(C)).decode('ascii'),
      "key": key.get_terms_list(),
      "structure": lie.get_structure_list()
   }
   with open("../encrypted.min.json", "w") as f:
      json.dump(exchange_data, f)
```

### 4.11 Основная программа `decrypt.py`

```python
def main():
    # Загрузка данных из JSON
    with open("encrypted.min.json", "r") as f:
        data = json.load(f)

    dim = data["dim"]

    # Восстановление алгебры Ли
    lie = LieAlgebra.from_structure_list(data["structure"], dim)
    set_rules(lie.get_reduction_rules())

    # Восстановление ключа
    key = Polynomial.from_terms_list(data["key"], dim)

    # Дешифрование
    C = bytes_to_polynomial(base64.b64decode(data["ciphertext"]), dim)
    q, r = C.reduce_full_fast(key)

    # Восстановление байтов
    recovered_bytes = polynomial_to_bytes(q, dim)

    # Сохранение в разных форматах
    with open("decrypted.bin", "wb") as f:
        f.write(recovered_bytes)

    # Попытка декодирования как UTF-8
    try:
        text = recovered_bytes.decode('utf-8')
        with open("decrypted.txt", "w", encoding="utf-8") as f:
            f.write(text)
    except UnicodeDecodeError:
        # Сохранение в base64 для диагностики
        with open("decrypted.b64", "w") as f:
            f.write(base64.b64encode(recovered_bytes).decode('ascii'))
```

## Криптоанализ и стойкость

Стойкость данной криптосистемы базируется на сложности задачи нахождения левого частного без знания ключа, что можно рассматривать как вариант задачи **дискретного логарифма в некоммутативном кольце**. В работе [1] предлагается использовать такие системы как потенциально пост-квантовые, поскольку для атак на некоммутативные структуры неизвестны эффективные квантовые алгоритмы.

### Факторы, влияющие на стойкость:

1. **Размерность алгебры Ли** — чем выше размерность, тем сложнее восстановить ключ
2. **Ступень нильпотентности** — влияет на сложность редукции
3. **Сложность ключа** — количество мономов и их степени
4. **Случайность структурных констант** — обеспечивает уникальность алгебры

### Известные уязвимости:

- Система является симметричной (один ключ для шифрования и дешифрования)
- Существует значительное расширение данных (шифротекст в 3-5 раз больше открытого текста)
- Отсутствует формальное доказательство стойкости; требуются дополнительные криптоаналитические исследования

## Заключение

В работе представлена реализация криптосистемы на основе универсальной обёртывающей алгебры Ли с автоматическим определением параметров. Система полностью работоспособна и корректно шифрует/дешифрует текстовые сообщения любой длины.

**Основные достижения:**
1. Автоматический выбор оптимальной размерности алгебры Ли
2. Динамическое распределение переменных для кодирования позиции
3. Эффективные алгоритмы умножения с кэшированием
4. Быстрое деление с использованием кучи (O(n log n))
5. Поддержка текстов произвольной длины

Математическая база опирается на теорию алгебр Ли, базисы Грёбнера–Ширшова и теорему Пуанкаре–Биркгофа–Витта. Реализация на Python демонстрирует принципиальную возможность практического использования подобных схем в пост-квантовой криптографии.
